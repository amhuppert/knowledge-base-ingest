You are an independent senior architect reviewing a greenfield design. Produce a complete, opinionated architecture. Be concrete and specific; prefer decisions over surveys. This is a DESIGN deliverable (no code yet).

# Problem
Build a system for using an AI coding agent (Claude Code) to CREATE, MAINTAIN, and INGEST INTO a knowledge base from source materials. Local, single-user. TypeScript + Zod + SQLite. CLI tools that AI agents invoke to find/update info efficiently. Delivered partly as Claude Code "skills" (markdown instruction files that guide the agent).

# Hard requirements
1. Given source materials, the agent creates a set of documents that are the current, complete, up-to-date view of the knowledge base.
2. Hierarchical summarization & synthesis: a top-level document, broken into a hierarchy of arbitrary depth. Parent documents are the synthesis of all their child documents.
3. An index document linking all source documents with short descriptions.
4. Copies of source documents kept in the KB, IMMUTABLE and segregated from anything derived/generated.
5. Strict PROVENANCE: every part of generated info traces back to a source document.
6. After initial creation, the system ingests NEW source documents and must update the generated KB docs to incorporate new info (including conflicts/supersession).
7. During ingestion, extract knowledge-graph items: entities of different types and relationships between them.
8. Goals: agents can use the KB to answer questions about the subject matter; system maintains human-readable documents.
9. V1 outcomes: humans get a readable-markdown view of the KB; Claude Code can efficiently and accurately find info and answer questions.

# Optimize for (handle well): technical documentation, software development topics, software engineering principles. Design for greater generality later.

# Rules: prioritize type safety; TDD red/green/refactor.

C. HIERARCHICAL SYNTHESIS mechanics: how the tree is stored & represented on disk and in DB; how "parent = synthesis of children" is enforced/checked; how nodes split when too big; how staleness propagates upward after an ingest.
